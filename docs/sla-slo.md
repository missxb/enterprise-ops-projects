# SLA/SLO定义

## 服务等级目标(SLO)
| 服务 | 可用性SLO | 月停机上限 | 延迟SLO | 数据持久性 |
|------|-----------|------------|---------|------------|
| K8s集群 | 99.95% | 21.9分钟 | P99<100ms | - |
| MySQL | 99.99% | 4.3分钟 | P99<50ms | 99.9999% |
| Redis | 99.99% | 4.3分钟 | P99<5ms | 99.999% |
| ES | 99.9% | 43.8分钟 | P99<200ms | 99.9999% |
| Harbor | 99.9% | 43.8分钟 | P99<500ms | 99.9999% |
| 监控 | 99.9% | 43.8分钟 | P99<1s | 99.99% |

## 错误预算
- 月错误预算 = (1 - SLO) × 本月分钟数
- K8s: 0.05% × 43200 = 21.6分钟
- MySQL: 0.01% × 43200 = 4.32分钟
- Redis: 0.01% × 43200 = 4.32分钟

## SLA违约赔偿
- 可用性<99.9%: 赔偿当月费用10%
- 可用性<99%: 赔偿当月费用30%
- 可用性<95%: 赔偿当月费用100%

## 监控指标
- 可用性: up指标成功/总请求数
- 延迟: P50/P90/P99响应时间
- 错误率: 5xx/总请求数
- 吞吐量: QPS/TPS

## 告警阈值
- SLO消耗>50%: 警告
- SLO消耗>80%: 严重
- SLO消耗>100%: 紧急(触发回滚)
